## 2. KPT 작성

### Keep: 스프린트 완료 후에도 간직하고 싶은 잘했던 것 / 좋았던 것

- 프로젝트 기획 명세 작성과 회의를 통해 실제 개발하게될 내용에 대해 더 잘 이해하게 될 수 있어서 좋았다.

### Problem: 스프린트 중 겪었던 어려움(기술, 소통, 협업, 에러 등 프로젝트 진행 관련된 그 어느것이든) / 스프린트 완료 후에도 아쉬움으로 남는 것

- 크게 어려움이나 아쉬움을 느낀 부분은 없었다.
